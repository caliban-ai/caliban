## Task 4: TDD — `Tool::invoke` happy paths

**Files:** modify `crates/caliban-tools-builtin/src/web_fetch.rs`.

`wiremock` setup pattern (factored into a test helper):

```rust
async fn mock_server() -> wiremock::MockServer { wiremock::MockServer::start().await }
fn url_of(m: &MockServer, path: &str) -> String { format!("{}{}", m.uri(), path) }
fn ctx() -> ToolContext { ToolContext { tool_use_id: "t1".into(), cancel: CancellationToken::new() } }
fn tool() -> WebFetchTool {
    WebFetchTool::new(
        reqwest::Client::builder()
            .redirect(reqwest::redirect::Policy::none())
            .build()
            .unwrap(),
    )
}
```

- [ ] **Step 1 (RED)** — write these tests, calling `tool.invoke(json!({"url": url_of(&server, "/")}), ctx()).await`:
  - `fetches_text_html_and_converts_to_markdown`
  - `fetches_text_plain_passthrough`
  - `fetches_json_passthrough`
  - `binary_content_returns_notice`
  - `non_2xx_returns_tool_error_with_status`

- [ ] **Step 2 (GREEN)** — implement `Tool::invoke`:
  1. Parse + validate input (`url`, optional `prompt`, optional `timeout_seconds`).
  2. Call `fetch_with_redirects(url, timeout, cx.cancel)` — returns `FetchOutcome` enum: `Ok { final_url, status, content_type, body_bytes }` or `Redirect { from, to, status }` or various errors.
  3. Classify content-type → render body (HTML→markdown via spawn_blocking, text passthrough, or binary notice).
  4. Optionally truncate.
  5. Format the header line + body.
  6. Return one `ContentBlock::Text`.

  The `fetch_with_redirects` loop:
  - For each hop (≤ 10): issue `client.get(current_url).send().await` inside `tokio::select!(cancel, timeout, send)`.
  - Status 301/302/307/308 → consult `Location` header.
  - If `same_host_ignoring_www(current_url, location)` → continue loop with `current_url = location`.
  - Else → return `Redirect` with original + location.
  - Otherwise → read body bounded; return `Ok`.

  Body reader:
  ```rust
  let mut acc = Vec::with_capacity(8192);
  while let Some(chunk) = resp.chunk().await? {
      if acc.len() + chunk.len() > MAX_BYTES { return Err(...) }
      acc.extend_from_slice(&chunk);
  }
  ```

- [ ] **Step 3 (REFACTOR)** — extract content-type classification + body-rendering branches into helpers. Confirm tests pass.

## Notes

- **Mock provider for summarizer tests.** Use `caliban_provider::mock::MockProvider`. Wrap in `Arc<dyn Provider + Send + Sync>` via `Arc::new(mock) as Arc<dyn Provider + Send + Sync>`. Enqueue a `CompletionResponse` with a text content block; the test asserts the tool output contains that text. For the failure-fallback test, enqueue `Err(Error::...)`.
- **`htmd::HtmlToMarkdown` thread-safety.** If it turns out NOT to be `Send + Sync`, the `OnceLock` refactor (Task 3 Step 3) is rolled back to building a fresh one per call (still cheap). Don't block on it.
- **`reqwest::Client` reuse.** The tool builds against an injected client. Tests build their own; the binary should build one shared client.
- **Header line `bytes` value.** For binary/redirect outputs, the `bytes` field is the raw body size received (could be 0 for redirect responses; that's fine).
