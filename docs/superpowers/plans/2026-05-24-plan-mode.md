# Plan Mode Implementation Plan

> Executed inline. Plan kept short for the record.

**Goal:** Add a read-only "plan mode" the model + operator can toggle to share a plan before mutating work. While active, the dispatcher rejects non-allowlisted tools before they invoke.

**Architecture:**
- `caliban_agent_core::plan_mode` exposes `SharedPlanMode = Arc<AtomicBool>` + `PLAN_MODE_ALLOWLIST` + `is_allowed_in_plan_mode(&str) -> bool`.
- `Agent.plan_mode: Option<SharedPlanMode>` (built via `AgentBuilder::plan_mode(handle)`); dispatcher in `stream.rs` checks the flag in Phase-1 of the dispatch loop and synthesizes a denied `ToolResult` for non-allowlisted tools.
- Two built-in tools, `EnterPlanModeTool` / `ExitPlanModeTool`, each own a clone of the shared flag.
- `PersistedSession.plan_mode: bool` with `#[serde(default)]` so legacy sessions resume with `false`.
- TUI: `/plan` slash command toggles the flag, mirrors it onto `session.plan_mode`, prints an Info line. Status bar grows a `📋 plan` chip when active.

**Tests delivered:**
- 4 unit tests on `plan_mode` module (default, atomic clone, allowlist contents, allowlist negative cases).
- 4 unit tests on the two tools (enter sets flag + echoes, exit clears, exit rejects `confirm=false`, exit defaults `confirm=true`).
- 3 integration tests in `tests/plan_mode_gating.rs` (mutating-tool blocked, read-only tool allowed, plan-mode-off lets mutating tools through).
