# Skills v1 Implementation Plan

> Executed inline.

**Goal:** Load YAML-frontmatter `SKILL.md` files from standard locations and expose them via a `Skill` tool the model can invoke by name.

**Architecture:**
- New `crates/caliban-skills` workspace member:
  - `skill.rs` — `Skill { name, description, body, metadata, source_path }`.
  - `loader.rs` — `default_roots(workspace) → [<ws>/.caliban/skills, $XDG_CONFIG_HOME/caliban/skills, $XDG_DATA_HOME/caliban/plugins]`; `load_skills(&[PathBuf]) -> Vec<Skill>` walks each root using `ignore::WalkBuilder`, parses YAML frontmatter, drops bad files with `warn!`, dedups by name with first-root-wins.
  - `tool.rs` — `SkillTool` impl `Tool`. Constructor builds a description listing every loaded skill (capped at 8 KiB with per-entry truncation marker). `invoke(name)` returns the matched skill's body as a single TextBlock or errors `InvalidInput` with a 10-name available preview.
- Binary wiring: `build_registry` registers `SkillTool::new(load_skills(&default_roots(&ws_root)))` after the other tools. `--no-skills` (env `CALIBAN_NO_SKILLS`) skips registration.
- TUI `/skills` slash command re-loads from disk and prints summary to the transcript. The old `Overlay::Skills` stub is retained for slash-menu enum parity but routed past.

**Tests delivered (10 in `tests/end_to_end.rs`):**
- loads_well_formed_skill, rejects_missing_frontmatter, rejects_mismatched_name, priority_first_root_shadows_later, missing_discovery_root_is_ok, skill_tool_returns_body_on_match, skill_tool_invalid_input_on_miss, skill_tool_description_lists_all_skills, empty_skill_tool_has_no_skills_in_description, metadata_passthrough.
