# Sub-agent (`AgentTool`) v1 Implementation Plan

> Executed inline.

**Goal:** Built-in `AgentTool` that spawns a synchronous in-process sub-agent with a restricted tool palette and returns a single text result to the parent.

**Architecture:**
- `crates/caliban-tools-builtin/src/agent_tool.rs` owns `AgentTool`, `AgentToolInput`, `AgentFactory`.
- Factory is a `Arc<dyn Fn(&AgentToolInput) -> Agent + Send + Sync>` closing over the parent provider, a snapshot of the parent's tool registry (already filtered to exclude `AgentTool`), and the parent model/max_tokens.
- `invoke` builds a fresh message vec (system prompt + user prompt), drives `stream_until_done` on a child cancellation token, collects the last assistant message's text, truncates at 5000 chars, returns one `ContentBlock::Text`.
- Cancellation: `cx.cancel.child_token()` ensures parent cancel propagates; sub-agent cannot cancel parent.
- Recursion guard: registry snapshot is taken BEFORE `AgentTool` is registered into the parent registry, so the snapshot can't contain `AgentTool`. Allowlist filtering also drops `"AgentTool"` defensively.
- `max_turns = 20` (hardcoded). On `MaxTurnsReached`, output is prefixed with `[sub-agent exhausted max_turns without completing]`.
- v1 limitations (deferred):
  - Sub-agent uses `NoopHooks`, not the parent's hooks (so permissions don't inherit). Documented.
  - No model-override against a router (we don't have one).
  - No memoization or budgeting across calls.

**Tests delivered (5):**
- Returns final text to parent.
- Truncates long output with `[sub-agent output truncated]` footer.
- Model override is honored (factory sees `input.model`).
- Cancellation propagates without panic.
- Invalid input (missing `prompt`) errors with `ToolError::InvalidInput`.

**CLI:**
- `--no-sub-agent` (env `CALIBAN_NO_SUB_AGENT`) — skip registering `AgentTool`.

**Files:**
- create `crates/caliban-tools-builtin/src/agent_tool.rs`
